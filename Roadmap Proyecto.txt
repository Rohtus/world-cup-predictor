WORLD CUP PREDICTOR 2026 — ROADMAP V1

Estado actual: El proyecto completó la primera etapa de Machine
Learning.

Se construyó el pipeline base:

Events ↓ Player Vector ↓ Team Vector ↓ Match Vector ↓ Feature
Engineering ↓ Model Comparison ↓ Feature Selection ↓ XGBoost Baseline ↓
XGBoost Optimization

Conclusión actual: El mejor modelo obtenido hasta ahora es Random Forest
V03, con accuracy aproximada de 0.515625 sobre el set de prueba 2022.

La optimización de XGBoost mejoró su rendimiento respecto al baseline,
pero no logró superar al Random Forest.

A partir de ahora, el proyecto cambia de foco: de optimizar modelos
históricos a construir un sistema capaz de predecir partidos futuros
usando planteles actuales.

==================================================

OBJETIVO V1

Construir una primera versión funcional del World Cup Predictor 2026
que:

1.  Use datos históricos del Mundial 2018 y Mundial 2022 como base de
    entrenamiento.
2.  Construya una base internacional actualizada de jugadores.
3.  Asocie automáticamente los planteles actuales con dicha base.
4.  Expanda la base incorporando jugadores nuevos cuando sea necesario.
5.  Construya Team Vectors actuales.
6.  Genere predicciones de partidos futuros.
7.  Simule el Mundial 2026 mediante Monte Carlo.

==================================================

ESTRATEGIA GENERAL

La V1 se divide en dos grandes etapas.

ETAPA A — Representación de selecciones actuales

Objetivo: Utilizar el modelo histórico ya validado alimentándolo con
representaciones actuales de las selecciones.

Estado:

✔ Competitions Inventory ✔ International Player Database ✔ Current
Roster Matching (automatic pipeline) ⬜ Player Database Expansion ⬜
Current Team Vectors ⬜ Future Match Vector ⬜ Prediction Pipeline ⬜
Monte Carlo Simulation ⬜ World Cup Simulator

Pipeline esperado:

International Player Database ↓ Current Roster Matching ↓ Expanded
Player Database ↓ Current Team Vector ↓ Future Match Vector ↓ Feature
Engineering V03 ↓ Feature Selection V03 ↓ Random Forest V03 ↓ Prediction

==================================================

ETAPA B — Expandir el entrenamiento

Objetivo: Agregar nuevas competiciones internacionales al dataset de
entrenamiento y evaluar experimentalmente su impacto sobre el
rendimiento del modelo.

No se asume que más datos impliquen mejores resultados.

==================================================

PRINCIPIO METODOLÓGICO

Cada notebook responde una pregunta específica.

Cada etapa debe producir un artefacto reutilizable dentro del pipeline.

La prioridad de la V1 es construir un pipeline funcional, explicable,
modular y reproducible.

==================================================

VERSIÓN FINAL PRE-MUNDIAL

La primera versión final del proyecto debería permitir:

1.  Construir una base internacional de jugadores.
2.  Asociar planteles actuales.
3.  Expandir automáticamente la base cuando aparezcan jugadores nuevos.
4.  Construir Team Vectors actuales.
5.  Predecir partidos futuros.
6.  Simular el Mundial 2026.
7.  Obtener probabilidades de clasificación y campeonato.

==================================================

VERSIÓN DEFINITIVA POST-MUNDIAL

Después del Mundial 2026 el proyecto podrá reentrenarse utilizando una
base internacional más amplia, incluyendo nuevas competiciones y el
propio Mundial 2026 para evaluar distintas ventanas históricas.

==================================================

VERSIÓN 2 FUTURA

La V2 podrá convertirse en un sistema continuamente actualizable capaz
de incorporar nuevos partidos, recalcular Player Vectors, Team Vectors,
rankings y simulaciones para futuras competiciones.
